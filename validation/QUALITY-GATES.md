# LOGOS Quality Gates

Quality Gates define when an object is ready to move forward.

A LOGOS object should not move to the next stage only because it sounds interesting.

It must pass the right gate.

---

## Gate 1 — Clarity

The object is clear enough that another person can understand it without the original conversation.

Questions:

- Is the title clear?
- Is the object type clear?
- Is the meaning understandable?
- Is the old/new movement visible when relevant?

---

## Gate 2 — Structure

The object follows the expected format.

Questions:

- Does it have an ID?
- Does it have a status?
- Does it have a language?
- Does it have required fields?
- Does it follow naming conventions?

---

## Gate 3 — Traceability

The object can be connected to the system.

Questions:

- Does it link to related objects?
- Can we trace it backward?
- Can we trace it forward?
- Does it duplicate another object?

---

## Gate 4 — Meaning Strength

The object carries real meaning, not just wording.

Questions:

- Does it start from a human truth?
- Does it reveal a contradiction?
- Does it create a belief shift?
- Can the core idea survive without a product?

---

## Gate 5 — Ethical and Safety Check

The object should not push false certainty, unsupported claims or unnecessary pressure.

Questions:

- Is the person respected?
- Is the claim supportable?
- Is fear used carefully?
- Does the object avoid overpromising?

---

## Gate 6 — Testability

The object can be tested in reality.

Questions:

- Can it become a script, post, page or experiment?
- Are success signals clear?
- Can we collect metrics?
- Can we learn from it?

---

## Gate 7 — Reusability

The object is useful beyond one immediate output.

Questions:

- Can it generate more than one script?
- Can it support more than one format?
- Can it become a pattern?
- Could it work outside the current runtime?

---

## Maturity Mapping

```text
raw → Gate 1
candidate → Gate 2
Draft → Gate 3
Testing → Gate 4 + Gate 5
Validated → Gate 6
Law Candidate → Gate 7 + repeated evidence
LOGOS Law → all gates + evidence
```

---

## Final Rule

A weak but structured object is better than a beautiful but untraceable idea.

LOGOS values clarity, traceability and learning over isolated brilliance.
